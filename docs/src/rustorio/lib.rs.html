1//! [![github]](https://github.com/albertsgarde/rustorio)&ensp;[![crates-io]](https://crates.io/crates/rustorio)
2//!
3//! [github]: https://img.shields.io/badge/github-8da0cb?style=for-the-badge&labelColor=555555&logo=github
4//! [crates-io]: https://img.shields.io/badge/crates.io-fc8d62?style=for-the-badge&labelColor=555555&logo=rust
5//!
6//! The first game written _and played_ entirely in Rust's type system. Not just do you play by
7//! writing Rust code, the rules of the game are enforced by the Rust compiler! If
8//! you can write the program so it compiles and doesn't panic, you win!
9//!
10//! ## How to play
11//!
12//! 1. Install [Rust](https://www.rust-lang.org/tools/install). Specifically it's
13//!    important to have the entire rustup toolchain and cargo, all of which you get
14//!    automatically by following the instructions in the link.
15//! 2. Install `rustorio` by running `cargo +nightly install rustorio`.
16//! 3. Set up a new Rustorio project by running `rustorio setup <path>`, where
17//!    `<path>` is the directory you want to create the project in (defaults to
18//!    '.').
19//! 4. Under `src/bin/tutorial/` you will find a tutorial save. You can start by
20//!    playing that one.
21//! 5. Playing the game consists of filling out the `user_main` function in the
22//!    `main.rs` file in the save game folder created for you.
23//! 6. Run with `rustorio play <save name>`. This will compile and run your save. If
24//!    it compiles and completes without panicking, you win! It'll then tell you how
25//!    many ticks it took you to win.
26//!
27//! ### After the tutorial
28//! To play other game modes, run `rustorio new-game` and specify a game mode.
29//! Use `rustorio new-game --help` to see all available game modes.
30//!
31//! ## Rules
32//!
33//! The rules are enforced by the compiler. Only thing I'd say is to not remove the
34//! `#![forbid(unsafe_code)]` at the top of the `main.rs` file, as using unsafe code
35//! can bypass most of what the compiler enforces. If you think you've found a way
36//! to cheat the game that isn't caught by the compiler, please file an issue! Part
37//! of my interest in this project is whether I can actually rule out all possible
38//! cheating vectors using only the Rust compiler. I haven't found a way to cheat
39//! yet, but I'm sure others will be more creative than me!
40//!
41//! ## Help
42//!
43//! Documentation for the Rustorio library can be found
44//! [here](https://albertsgarde.github.io/rustorio/rustorio). A good place to start
45//! is to build a
46//! [furnace](https://albertsgarde.github.io/rustorio/rustorio/buildings/struct.Furnace.html)
47//! and start
48//! [mining](https://albertsgarde.github.io/rustorio/rustorio/fn.mine_iron.html) and
49//! smelting iron. Alternatively, you can work backwards by looking at the
50//! [recipe](https://albertsgarde.github.io/rustorio/rustorio/recipes/struct.PointRecipe.html)
51//! for points to figure out how to get them.

71static ONCE: Once = Once::new();

73/// Runs your play. If it is run multiple times, it will panic. This is to prevent using multiple threads to cheat.
74pub fn play<G: GameMode>(main: fn(Tick, G::StartingResources) -> (Tick, G::VictoryResources)) -> ! {
75    let mut call_once_ran = false;
76    ONCE.call_once(|| call_once_ran = true);
77    if !call_once_ran {
78        panic!("play() can only be called once per program execution to prevent cheating via multithreading.");
79    }
80    let tick = Tick::start();
81    let start_resources = G::StartingResources::init();
82    let (tick, _points) = main(tick, start_resources);
83    println!("You won in {} ticks!", tick.cur());
84    std::process::exit(0);
85}

87/// A bundle of specifically iron ore.
88pub type IronOreBundle<const AMOUNT: u32> = Bundle<{ ResourceType::IronOre }, AMOUNT>;
89
90/// Mines iron ore. Takes 2 ticks to mine 1 ore.
91pub fn mine_iron<const AMOUNT: u32>(tick: &mut Tick) -> IronOreBundle<AMOUNT> {
92    tick.advance_by(2 * AMOUNT as u64);
93    Bundle::new()
94}
